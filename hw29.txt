2. С помощью какого метода можно отловить изменение `props`?
shouldComponentUpdate(), componentDidUpdate() вызывается сразу после обновления. Не вызывается при первом рендере.

4. В каком из методов жизненного цикла лучше всего использовать методы вызова API и обращения к веб-хранилищам, если они должны быть вызваны всего один раз при загрузке страницы?
componentDidMount()

5. С помощью какого метода можно отловить и отрисовать для пользователя возникшую в компоненте ошибку?

6. Какой код обычно пишут в конструкторе? Для каких задач он используется?
Для инициализации состояния и каких-то обработчиков события

7. Что делает функция `render`()? Что может ее вызвать?
Рисует компонент. Это может быть на этапе создания компонента, перерисовка род.элемента, при изменении состояния, вызов функции forceUpdate, перехват ошибки.

8. Что нужно изменить в коде из урока, чтобы начальные параметры у компонента приходили из пропсов, но если пропсы вообще не заданы, начальные значения были инициализированы нулями?
В функциональном создать функцию Component.defaultProps, а в классовом прописать static defaultProps

9. Можно ли несколько раз использовать хук useEffect внутри одного компонента?
Да

10. Можно ли не передавать второй аргумент в хук useEffect, что  тогда произойдет?
Он необязательный, но его использование помогает пропустить выполнение эффекта, если определенные значения не изменились между повторными отрисовками. 
То есть если его убрать, то будет перерисовка, даже если ничего не поменялось.

11. Что означает возвращение функции в теле эффекта? 
это метод componentWillUnmount, который вызывается только один раз перед размонтированием компонента
То есть эта переданная функция выполнить только при сбрасывании эффекта
